# セキュリティレポート

アクティブなセキュリティポリシーの詳細については、[this page](https://github.com/nodejs/node/security/policy)をご覧ください。

## Node.js のバグの報告

[HackerOne](https://hackerone.com/nodejs) 経由で Node.js のセキュリティバグを報告します。

レポートは5日以内に承認されます。 そして、提出を処理する
の次のステップを示す10日以内に、レポートへの
詳細な返答を受け取ることができます。

レポートへの最初の返信後、セキュリティチームは、修正と完全な公表に向けた進捗状況をお知らせし、報告された問題に関する追加情報やガイダンスを求めることがあります。

### Node.js バグ報奨金プログラム

Node.jsプロジェクトは、セキュリティ研究者と責任ある公開開示のための公式のバグバウンティプログラムに参加しています。このプログラムはHackerOneプラットフォームを通じて管理されています。詳細については[https://hackerone.com/nodejs](https://hackerone.com/nodejs)を参照してください。

## サードパーティーモジュールのバグの報告

サードパーティモジュールのセキュリティバグは、それぞれの
メンテナに報告する必要があります。

## 開示方針

Node.js のセキュリティ開示ポリシーは次のとおりです。

* セキュリティレポートは受け取られ、主要なハンドラーに割り当てられます。この人物は修正とリリースプロセスを調整します。問題が確認され、影響を受けるすべてのバージョンのリストが決定されます。潜在的に類似の問題を見つけるためにコードが監査されます。まだメンテナンスされているすべてのリリースのために修正が準備されます。これらの修正は公共のリポジトリにコミットされるのではなく、公表を待つためにローカルに保持されます。

* 本脆弱性の禁輸日を選択し、脆弱性についてはCVE(Common
  Vulnerabilities and Exposure®️)が必要となります。

* 公表日には、Node.jsセキュリティメーリングリストに公表のコピーが送信されます。変更内容は公共のリポジトリにプッシュされ、新しいビルドがnodejs.orgに展開されます。メーリングリストが通知されてから6時間以内に、アドバイザリのコピーがNode.jsブログに公開されます。

* 通常、公表日はCVEが発行されてから72時間後に設定されます。ただし、バグの深刻度や修正の難しさによって異なる場合があります。

* このプロセスには時間がかかることがあります、特に他のプロジェクトのメンテナとの調整が必要な場合。可能な限り迅速にバグを処理する努力がされますが、一貫した方法で開示が処理されるようにするために、上記のリリースプロセスに従うことが重要です。
