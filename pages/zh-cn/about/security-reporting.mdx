# 安全报告

有关活跃安全策略的更多详细信息，请查看[此页面](https://github.com/nodejs/node/security/policy)。

## 在 Node.js 中报告一个错误

通过 \[HackerOne]报告Node.js 中的安全漏洞(https://hackerone.com/nodejs)。

您的报告将在5天内得到确认，并将在接下来的10天内收到更详细的回复，说明处理您的报告的下一步措施。

在对您的报告进行初步回复后，安全团队将努力让您了解修复工作的进展情况和完整公告，并可能会要求提供有关报告问题的额外信息或指导。

### Node.js漏洞赏金计划

Node.js 项目为安全研究人员和负责任的公开披露提供官方漏洞赏金计划。该计划通过 HackerOne 平台进行管理。更多详情请查看 [https://hackerone.com/nodejs](https://hackerone.com/nodejs)。

## 在第三方模块中报告一个错误

第三方模块中的安全漏洞应该报告给他们各自的维护者。

## 披露政策

下面是 Node.js 的安全披露政策

* 安全报告已收到并分配了主要处理人员。这个人将协调修复和发布流程。问题已确认，并确定了所有受影响版本的列表。对代码进行审计，以查找任何潜在的类似问题。为仍在维护中的所有版本准备修复程序。这些修复程序尚未提交到公共存储库，而是在公告发布前本地保存。

* 选择了此漏洞的建议禁运日期，并请求了漏洞的CVE（通用漏洞和漏洞披露（CVE®））。

* 在禁运日，Node.js安全邮寄名单将发送一份
  通知。 更改被推送到公共仓库，新版本
  被部署到节点。 rg. 在
  发出邮件列表后的6小时内，咨询的副本将在Node.js 博客上发表。

* 通常，禁运日期将从CVE发布之时起的72小时后设定。然而，这可能会因bug的严重程度或应用修复的难度而有所不同。

* 这个过程可能需要一些时间，特别是在需要与其他项目的维护者协调时。我们将尽一切努力以尽可能及时地处理bug；然而，重要的是我们遵循上述发布流程，以确保披露以一致的方式处理。
