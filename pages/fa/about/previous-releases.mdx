# نسخه‌های قبلی

نسخه‌های اصلی Node.js برای شش ماه وارد وضعیت انتشار *Current* می‌شوند که به نویسندگان کتابخانه زمان می‌دهد تا پشتیبانی را برای آن‌ها اضافه کنند.
پس از گذشت شش ماه، نسخه‌های فرد (9، 11 و غیره) پشتیبانی نمی‌شوند و نسخه‌های زوج (10، 12 و غیره) وارد وضعیت *Active LTS* می‌شوند و برای استفاده عمومی آماده می‌شوند.
وضعیت انتشار *LTS* به معنای "پشتیبانی بلندمدت" است که به طور معمول تضمین می‌کند که باگ‌های اساسی برای کلیه 30 ماه تصحیح خواهند شد.
برنامه‌های تولیدی فقط باید از نسخه‌های *Active LTS* یا *Maintenance LTS* استفاده کنند.
