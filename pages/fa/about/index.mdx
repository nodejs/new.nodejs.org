# درباره Node.js®

به عنوان یک زمان اجرای جاوااسکریپت ناهمگام وابسته به رویداد، Node.js برای ساخت برنامه های شبکه قابل مقیاس طراحی شده است. در مثال "سلام دنیا" زیر، اتصالات زیادی می توانند به صورت همزمان اداره شوند. با هر اتصال، بازخوانی انجام می شود، اما اگر کاری برای انجام وجود نداشته باشد، Node.js خوابیده خواهد شد.

```js
const http = require('node:http');

const hostname = '127.0.0.1';
const port = 3000;

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World');
});

server.listen(port, hostname, () => {
  console.log(`Server running at http://${hostname}:${port}/`);
});
```

این با مدل همزمانی رایج امروز متفاوت است که در آن رشته های سیستم عامل استفاده می شوند. شبکه مبتنی بر رشته نسبتاً ناکارآمد و بسیار دشوار برای استفاده است. علاوه بر این، کاربران Node.js از نگرانی های قفل کردن فرآیند آزاد هستند، زیرا هیچ قفلی وجود ندارد. تقریباً هیچ تابعی در Node.js به طور مستقیم I/O انجام نمی دهد، بنابراین فرآیند هرگز قفل نمی شود مگر اینکه I/O با استفاده از روش های همگانی کتابخانه استاندارد Node.js انجام شود. به دلیل عدم قفل شدن چیزی، سیستم های قابل مقیاس توسعه دادنی در Node.js بسیار منطقی هستند.

اگر برخی از این زبان ناآشنا باشد، یک مقاله کامل در مورد \[مسدود کردن در مقابل عدم مسدود کردن]\[] وجود دارد.

Node.js در طراحی خود شبیه به سیستم هایی مانند \[Event Machine]\[] Ruby و \[Twisted]\[] Python است. Node.js مدل رویداد را یکم بیشتر می بیند. این یک حلقه رویداد را به عنوان یک سازه زمان اجرا نشان می دهد به جای یک کتابخانه. در سایر سیستم ها، همیشه یک تماس مسدود کننده برای شروع حلقه رویداد وجود دارد. به طور معمول، رفتار از طریق بازخوانی ها در ابتدای یک اسکریپت تعریف می شود و در انتها یک سرور از طریق یک تماس مسدود مانند `EventMachine::run()` شروع می شود. در Node.js، چنین تماسی برای شروع حلقه رویداد وجود ندارد. Node.js به سادگی پس از اجرای اسکریپت ورودی وارد حلقه رویداد می شود. Node.js از حلقه رویداد خارج می شود زمانی که دیگر بازخوانی برای انجام وجود نداشته باشد. این رفتار مانند جاوااسکریپت مرورگر است - حلقه رویداد از کاربر مخفی است.

HTTP یک شهروند اول در Node.js است، طراحی شده با استریمینگ و کم‌تاخیر در ذهن. این باعث می‌شود Node.js مناسب برای پایه‌گذاری یک کتابخانه یا چارچوب وب باشد.

اینکه Node.js بدون رشته‌ها طراحی شده است به این معنا نیست که نمی‌توانید از چندین هسته در محیط خود استفاده کنید. فرآیندهای فرزندی می‌توانند با استفاده از API \[`child_process.fork()`]\[] ایجاد شوند و طراحی شده‌اند تا ارتباط با آنها آسان باشد. بر پایه همان رابط، ماژول \[`cluster`]\[] ساخته شده است که به شما امکان می‌دهد تا سوکت‌ها را بین فرآیندها به اشتراک بگذارید تا بارگذاری تعادلی را بر روی هسته‌های خود فعال کنید.
