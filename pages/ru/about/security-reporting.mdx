# Отчетность по безопасности

Для получения более подробной информации об активных политиках безопасности просмотрите [эту страницу](https://github.com/nodejs/node/security/policy).

## Сообщить об ошибке в Node.js

Сообщайте об ошибках безопасности в Node.js через [HackerOne](https://hackerone.com/nodejs).

Ваш отчет будет рассмотрен в течение 5 дней, и вы получите более подробный ответ на ваш отчет в течение 10 дней, указывающий следующие шаги по обработке вашей заявки.

После первого ответа на ваш отчет команда безопасности будет стараться информировать вас о прогрессе в работе над исправлением и полным объявлением, а также может попросить дополнительную информацию или руководство по сообщенной проблеме.

### Программа для багов Node.js

Проект Node.js участвует в официальной программе по вознаграждению за нахождение уязвимостей для исследователей безопасности и ответственных публичных раскрытий. Программа управляется через платформу HackerOne. См. [https://hackerone.com/nodejs](https://hackerone.com/nodejs) для дополнительной информации.

## Сообщить об ошибке в стороннем модуле

Ошибки безопасности в сторонних модулях должны быть доведены до сведения соответствующих
сопровождающих.

## Политика раскрытия информации

Вот политика раскрытия безопасности для Node.js

* Отчет о безопасности получен и назначен основной обработчик. Этот человек будет координировать процесс исправления и выпуска. Проблема подтверждена, и определяется список всех затронутых версий. Код проходит аудит для выявления любых потенциальных аналогичных проблем. Исправления готовятся для всех версий, которые все еще находятся под обслуживанием. Эти исправления не фиксируются в общем репозитории, а хранятся локально в ожидании объявления.

* Для этой уязвимости выбрана предлагаемая дата эмбарго и запрошена CVE (Common
  Vulnerabilities and Exposures (CVE®️)).

* В день блокады список рассылки Node.js по безопасности отправлен копию объявления
  . Изменения вносятся в публичный репозиторий и новые сборки
  развернуты на узлах. rg. В течение 6 часов с момента отправки рассылки
  уведомляется копия рекомендации будет опубликована в блоге Node.js.

* Обычно дата снятия секретности устанавливается через 72 часа с момента выдачи CVE. Однако это может измениться в зависимости от серьезности ошибки или сложности в применении исправления.

* Этот процесс может занять некоторое время, особенно когда требуется согласование с обслуживающими другие проекты. Будет приложено все усилие для обработки ошибки в наиболее своевременный манере; однако важно, чтобы мы следовали вышеуказанному процессу выпуска, чтобы обеспечить, что раскрытие обрабатывается последовательно.
